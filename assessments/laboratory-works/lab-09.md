## 2. ТЕОРЕТИЧЕСКАЯ ЧАСТЬ

### 2.1. Задача трекинга объектов

Трекинг (tracking) — это задача отслеживания одного и того же объекта на протяжении последовательности кадров видеопотока. В отличие от детекции, которая выполняется на каждом кадре независимо, трекинг использует информацию из предыдущих кадров для предсказания положения объекта и сохранения его идентичности

Основные задачи трекинга:

- Локализация — определение положения объекта в каждом кадре.
- Идентификация — присвоение уникального идентификатора (ID) каждому объекту.
- Сохранение идентичности — поддержание одного и того же ID для объекта при его перемещении, временном исчезновении или перекрытии с другими объектами.

### 2.2. Алгоритм SORT (Simple Online and Realtime Tracking)

SORT — это базовый алгоритм трекинга, основанный на следующих компонентах:

- Калмановский фильтр — предсказывает положение объекта в следующем кадре на основе его текущего состояния (позиция, скорость).
- Венгерский алгоритм — решает задачу назначения (matching) между предсказанными и обнаруженными объектами на основе метрики IoU (Intersection over Union).
- Управление треками — создание новых треков для не сопоставленных детекций и удаление старых треков, которые долго не обновлялись.

**Ограничения SORT:**

Использует только пространственную информацию (IoU) для сопоставления. Неэффективен при длительных перекрытиях объектов или их временном исчезновении.

### 2.3. Алгоритм DeepSORT (Deep Simple Online and Realtime Tracking)

DeepSORT расширяет SORT, добавляя глубинный дескриптор внешнего вида (appearance descriptor) для более надёжного сопоставления объектов.

Ключевые компоненты DeepSORT:

- Калмановский фильтр — предсказание состояния объекта (аналогично SORT).
- Глубинная свёрточная нейронная сеть — извлечение вектора (дескриптора внешнего вида) для каждого обнаруженного объекта.
- Метрика косинусного сходства — сравнение дескрипторов внешнего вида для сопоставления объектов.
- Каскадное сопоставление — приоритетное сопоставление объектов с высокой уверенностью.

Преимущества DeepSORT:

- Устойчивость к перекрытиям и временным исчезновениям объектов.
- Возможность повторной идентификации объекта после его потери
- Более высокая точность трекинга в сложных сценариях.

### 2.4. Подсчёт объектов и анализ траекторий

Подсчёт уникальных объектов — одна из ключевых задач видеоаналитики:

- Каждый объект получает уникальный track ID, который сохраняется на протяжении всего времени его присутствия в кадре
- Для подсчёта используется множество (set) уникальных ID, появившихся в видеопотоке.
- Для подсчёта пересечений виртуальных линий используется анализ траектории: направление движения определяется по изменению координат объекта между кадрами.

Анализ траекторий включает:

- Определение направления движения (вход/выход из зоны).
- Построение карты траекторий.
- Вычисление скорости и времени нахождения в зоне.

### 3.3. Реализация базового пайплайна с трекингом

**Задание 2:**

- Реализуйте функцию, которая:
  - Читает видеофайл покадрово.
  - Для каждого кадра выполняет детекцию объектов с помощью YOLO.
  - Передаёт результаты детекции в DeepSORT для обновления треков.
  - Визуализирует bounding boxes с уникальными ID объектов.
  - Записывает обработанный кадр в выходной видеофайл.
- Базовый шаблон кода:

```python
def process_video_with_tracking(input_path, output_path, model, tracker, conf_threshold=0.25):
    cap = cv2.VideoCapture(input_path)
    
    fps = int(cap.get(cv2.CAP_PROP_FPS))
    width = int(cap.get(cv2.CAP_PROP_FRAME_WIDTH))
    height = int(cap.get(cv2.CAP_PROP_FRAME_HEIGHT))
    
    fourcc = cv2.VideoWriter_fourcc(*'XVID')
    out = cv2.VideoWriter(output_path, fourcc, fps, (width, height))
    
    frame_count = 0
    total_time = 0
    
    while cap.isOpened():
        ret, frame = cap.read()
        if not ret:
            break
        
        start_time = time.time()
        
        # Детекция объектов
        detections = model(frame, conf=conf_threshold)[0]
        
        # Подготовка данных для трекера: [x1, y1, x2, y2, confidence, class]
        dets = []
        for box in detections.boxes:
            x1, y1, x2, y2 = map(int, box.xyxy[0])
            conf = float(box.conf[0])
            cls = int(box.cls[0])
            dets.append(([x1, y1, x2, y2], conf, cls))
        
        # Обновление треков
        tracks = tracker.update_tracks(dets, frame=frame)
        
        # Визуализация
        for track in tracks:
            if not track.is_confirmed():
                continue
            track_id = track.track_id
            ltrb = track.to_ltrb()
            x1, y1, x2, y2 = map(int, ltrb)
            cls = track.det_class
            
            # Отрисовка bounding box и ID
            cv2.rectangle(frame, (x1, y1), (x2, y2), (0, 255, 0), 2)
            cv2.putText(frame, f'ID: {track_id}', (x1, y1 - 10),
                        cv2.FONT_HERSHEY_SIMPLEX, 0.5, (0, 255, 0), 2)
        
        out.write(frame)
        frame_count += 1
        total_time += time.time() - start_time
        
        if frame_count % 30 == 0:
            avg_fps = frame_count / total_time
            print(f'Кадров: {frame_count}, Средний FPS: {avg_fps:.2f}')
    
    cap.release()
    out.release()
    print(f'Обработка завершена. Всего кадров: {frame_count}')
    print(f'Средний FPS: {frame_count / total_time:.2f}')
```

**Методические указания:**

Для получения данных детекции используйте `detections.boxes`.

Трекер DeepSORT возвращает объекты `Track`, содержащие `track_id`, координаты bounding box (`to_ltrb()`) и класс объекта (`det_class`).
