## 2. ТЕОРЕТИЧЕСКАЯ ЧАСТЬ

### 2.1. Основные этапы работы YOLO:

- Входное изображение разделяется на сетку ячеек (grid).
- Для каждой ячейки предсказываются bounding boxes, уверенность (confidence score) и вероятности классов.
- Применяется постобработка для фильтрации и уточнения предсказаний.

### 2.2. Ключевые понятия инференса

**Confidence Score (оценка уверенности)** — вероятность того, что в предсказанном bounding box действительно содержится объект, и что предсказание корректно. Значение находится в диапазоне [0, 1]. Чем выше уверенность, тем более «надёжным» считается обнаружение.

**Порог уверенности (confidence threshold)** — минимальное значение уверенности, при котором детекция сохраняется в выходных результатах. Слишком низкий порог приводит к большому количеству ложных срабатываний (false positives), слишком высокий — к пропуску реальных объектов (false negatives).

**Non-Maximum Suppression (NMS)** — алгоритм постобработки, удаляющий дублирующиеся bounding boxes для одного и того же объекта. Принцип работы:

- Все предсказания сортируются по убыванию уверенности.
- Выбирается box с максимальной уверенностью, все остальные box'ы с пересечением (IoU) выше заданного порога подавляются (удаляются).
- Процесс повторяется для оставшихся box'ов.

### 2.3. Метрики оценки качества детекции

- mAP (mean Average Precision) — основная метрика для задач детекции, усреднённая по всем классам. Учитывает как точность (precision), так и полноту (recall) при разных порогах IoU.
- IoU — используется для оценки точности локализации.
- FPS (frames per second) — метрика производительности, важная для систем реального времени.

```python
for box in results[0].boxes:
```
