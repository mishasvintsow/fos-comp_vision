# Примеры тестовых материалов

Материалы служат примерами для составления теста, а не защищённым экзаменационным банком. Поэтому вопросы и ключи намеренно хранятся рядом в отдельных файлах. Университет-реализатор самостоятельно утверждает фактический набор заданий, варианты и режим доступа.

При подготовке фактического теста преподаватель:

1. проверяет корректность ключа;
2. соотносит вопрос с одним модулем и индикатором действующей РПД;
3. исключает задания, ответ на которые зависит от неуказанных условий;
4. проверяет сохранность формул и обозначений;
5. формирует вариант по спецификации из 30 заданий.

Тест измеряет теоретическую подготовку и способность выбирать методы. Практические результаты подтверждаются лабораторными и проектом.

## Рекомендованные промпты для расширения банка

Промпты предназначены для подготовки черновиков. Сгенерированные задания и ключи обязательно проходят предметную проверку преподавателем.

### Создание дополнительного блока

```text
Ты — преподаватель дисциплины «Методы компьютерного зрения».

Подготовь 10 тестовых заданий для:
- модуля: [название и содержание модуля];
- индикатора КРМ: [код и формулировка];
- уровня: [базовый / средний / продвинутый].

Требования:
1. Проверяй понимание и выбор метода, а не запоминание названий.
2. Используй практические ситуации из компьютерного зрения.
3. Для каждого задания предложи четыре варианта A–D.
4. Явно укажи, один или несколько ответов являются правильными.
5. Исключи варианты, правильность которых зависит от неуказанных условий.
6. Не используй формулировки «всегда», «никогда» и «лучший» без заданного критерия.
7. После заданий приведи отдельный ключ с кратким обоснованием каждого ответа.
8. Не копируй формулировки из переданного примера; сохрани только его сложность и структуру.

Содержание модуля:
[вставить строки тематического плана и относящиеся к ним дескрипторы из матрицы]
```

### Проверка подготовленного блока

```text
Проведи предметную экспертизу тестового блока по компьютерному зрению.

Для каждого вопроса проверь:
- соответствие указанным модулю, индикатору и уровню;
- однозначность условия;
- полноту исходных данных;
- число фактически правильных вариантов;
- корректность ключа;
- отсутствие подсказок в формулировках вариантов;
- актуальность терминов и API.

Верни таблицу:
«номер — проблема — критичность — предлагаемая редакция — проверенный ключ».
Не изменяй вопрос, если существенных замечаний нет. Все сведения о конкретных библиотеках
сверяй с их официальной документацией.

Проверяемый блок:
[вставить вопросы и предварительный ключ]
```
