## Профили вычислительных ресурсов

### Минимальный профиль

- компьютер с 4 вычислительными ядрами, 16 ГБ оперативной памяти и не менее 50 ГБ свободного места;
- стабильный доступ к сети для получения открытых наборов данных, документации и весов моделей;
- доступ к общей или облачной GPU-среде не менее чем с 8 ГБ видеопамяти для ресурсоёмких частей лабораторных работ № 7–9;
- возможность локального запуска Python, Jupyter и используемой среды разработки.

Наличие дискретного GPU у каждого обучающегося не является обязательным. Университет-реализатор должен предоставить общий вычислительный ресурс либо заранее согласованный облегчённый вариант задания, сохраняющий проверяемый результат обучения.

### Рекомендуемый профиль

- 8 вычислительных ядер, 32 ГБ оперативной памяти и не менее 100 ГБ свободного места;
- NVIDIA GPU с 16 ГБ видеопамяти либо сопоставимый по возможностям облачный ускоритель;
- Linux-среда или Windows с WSL2 для компонентов, чувствительных к операционной системе;
- общий репозиторий данных и весов моделей, чтобы исключить многократную загрузку одних и тех же ресурсов.

## Ресурсы по оценочным материалам

### LAB-01

| Компонент | Требование |
|---|---|
| Входные данные | Небольшой набор растровых изображений, включая изображения для проверки геометрических преобразований и фильтрации |
| Средства | Python, NumPy, OpenCV, Matplotlib |
| Инфраструктура | CPU, 8–16 ГБ оперативной памяти; GPU не требуется |
| Рекомендуемая литература | Матвеев; Селянкин; Klette; Szeliski |

### LAB-02

| Компонент | Требование |
|---|---|
| Входные данные | MNIST или другой сопоставимый открытый набор изображений |
| Средства | Python, NumPy, одна из поддерживаемых библиотек глубокого обучения |
| Инфраструктура | CPU и 16 ГБ оперативной памяти; GPU опционален |
| Рекомендуемая литература | Дэвис, Тёрк; Szeliski |

### LAB-03

| Компонент | Требование |
|---|---|
| Входные данные | Открытый набор изображений для многоклассовой классификации |
| Средства | Python, TensorFlow/Keras, средства визуализации |
| Модели | Предобученная CNN-архитектура, допускающая перенос обучения |
| Инфраструктура | CPU допустим для сокращённой выборки; общая GPU-среда рекомендуется |

### LAB-04

| Компонент | Требование |
|---|---|
| Входные данные | Размеченный по классам набор изображений в каталогах обучающей и проверочной выборок |
| Средства | Python, PyTorch или сопоставимая библиотека |
| Модели | Предобученная CNN-архитектура и модифицируемая классификационная голова |
| Инфраструктура | GPU от 6–8 ГБ видеопамяти рекомендуется; допускается сокращение числа эпох и объёма выборки |

### LAB-05

| Компонент | Требование |
|---|---|
| Входные данные | Синтетический набор для классификации и небольшой открытый набор для детекции, например COCO128 |
| Средства | Python, scikit-learn, PyTorch, Ultralytics YOLO или сопоставимые средства |
| Инфраструктура | CPU достаточен для базовых экспериментов; GPU ускоряет обучение детектора |
| Рекомендуемая литература | Дэвис, Тёрк; Szeliski |

### LAB-06

| Компонент | Требование |
|---|---|
| Входные данные | Не менее 10 размеченных изображений выбранной предметной области |
| Средства | Инструмент разметки, Python, библиотека детекции объектов |
| Модели | Предобученные веса детектора для дообучения |
| Инфраструктура | GPU рекомендуется; при отсутствии используется малая модель и сокращённый объём обучения |

### LAB-07

| Компонент | Требование |
|---|---|
| Входные данные | MNIST; для дополнительного сравнения — Fashion-MNIST |
| Средства | Python, TensorFlow/Keras, NumPy, Matplotlib |
| Модели | Базовая GAN и DCGAN |
| Инфраструктура | Локальный CPU допустим для сокращённого обучения; общая GPU-среда от 6–8 ГБ видеопамяти рекомендуется |

### LAB-08

| Компонент | Требование |
|---|---|
| Входные данные | Текстовые описания, исходные изображения и маски для управляемой генерации |
| Средства | Python, PyTorch, Diffusers, Transformers, Accelerate |
| Модели | Согласованная версия Stable Diffusion, модель инпейнтинга и ControlNet |
| Минимальная инфраструктура | Общая GPU-среда от 8 ГБ видеопамяти при использовании облегчённой модели и оптимизаций памяти |
| Рекомендуемая инфраструктура | GPU с 16 ГБ видеопамяти; для более крупных моделей — ресурс, соответствующий требованиям выбранной версии |
| Альтернатива | Предоставленная университетом облачная среда или заранее согласованная облегчённая модель; вариант должен сохранять генерацию, инпейнтинг и управляемое условие |

### LAB-09

| Компонент | Требование |
|---|---|
| Входные данные | Короткий видеоролик с объектами выбранных классов; при необходимости — эталонная разметка фрагмента |
| Средства | Python, OpenCV, YOLO, DeepSORT, PySpark, Apache Airflow |
| Инфраструктура | 4 ядра CPU, 16 ГБ оперативной памяти; GPU опционален для детекции и трекинга |
| Среда обработки | Локальный режим PySpark и локальная либо контейнерная среда Airflow, если кластер не предоставлен |
| Ограничение объёма | Для минимального профиля используется короткое видео и малая модель детектора |

<a id="project"></a>
### Проектная работа

- [универсальный шаблон проектного кейса](../assessments/project/README.md);
- открытый или предоставленный индустриальным партнёром набор данных, доступный на законных основаниях;
- вычислительный ресурс, достаточный для выбранного масштаба решения;
- документация используемых моделей, библиотек и метрик;
- источники для сопоставления результата с базовым или современным подходом из [каталога научных статей](papers/README.md).

До выдачи кейса университет-реализатор заполняет переменные поля шаблона, фиксирует доступ к данным и инфраструктуре, а также допустимые замены недоступных ресурсов.
